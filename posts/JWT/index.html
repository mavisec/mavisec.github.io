🔓 Cracking Open JWTs (Ethically): How to Decode, Inspect, and Test Them with jwt.io

2025-08-23
#JWT  #WebSecurity  #Pentesting 

JSON Web Tokens (JWTs) are everywhere—powering logins, APIs, and sessions. As a hacker or pentester, learning how to decode, inspect, and test them is essential. In this post, we’ll crack open JWTs the ethical way, using jwt.io.

🔹 What is a JWT?

A JWT is split into three parts separated by dots:

<header>.<payload>.<signature>
  • Header: contains metadata like algorithm (alg) and type (typ).
  • Payload: contains claims (e.g., username, roles, expiration). Remember—it’s encoded, not encrypted.
  • Signature: prevents tampering by verifying integrity.

🔹 Decoding with jwt.io

The fastest way to decode JWTs is with jwt.io. Just paste your token and instantly see:

  • The header (algorithm, type).
  • The payload (claims like user info, roles, expiry).
  • How the signature changes when you modify data.

🔹 Inspecting JWTs (The Hacker’s Angle)

Why bother decoding JWTs? Because they can reveal:

  • Hidden user details like "role": "admin".
  • Expiration times (exp).
  • Weak or misconfigured algorithms (like alg: none).

As a pentester, just reading the payload can give you valuable recon data about the app you’re testing.

🔹 Simple Tests You Can Try

  • Change claims: Edit "role":"user" to "role":"admin" and see if the app accepts it.
  • Check expiry: Test if expired tokens are still valid.
  • Look for weak configs: If you see alg: none, that’s a red flag.

Note: Always do this ethically, in labs or with permission.

🔹 Defensive Tips

Because this is about ethical hacking, here’s how devs should defend:

  • Always validate signatures server-side.
  • Use strong secrets or proper key pairs.
  • Never allow alg: none.
  • Keep tokens short-lived and refresh securely.

🔹 Key Takeaways

  • JWTs are common in modern apps.
  • They’re encoded, not encrypted—so decoding is trivial.
  • jwt.io makes exploring tokens fast and easy.
  • Even beginners can test for simple issues like role manipulation or weak algorithms.
